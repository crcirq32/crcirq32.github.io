Introduction to Network Security

Moving into a new residence with network and hardware vulnerabilities exacerbated the need to learn more about IT and NetSec. With no prior knowledge of internet networking, all the protocols seem daughting. UDP, ICMP, TCP, SSH, IPv4/IPV6, IPSec, DNS, Proxy, Reverse Proxies, where does it end? Does it even end? Do not fret, let's simplify. The table below gives a brief overview of the OSI Model or the Open Systems Interconnection Model. These layers are where all computer communication takes place.

Understanding the flow of internet traffic is great for troubleshooting problems in the network. In order for computers to commiuncate with each other they need to have a unique identifier, a way for each device to be distinguished separately. These unique identifiers are assigned by a router provided by your ISP (internet service provider). All network traffic needs to be filtered, generally, through one location. The router catalogs every device in promixity with an IPv4 or IPv6 IP address. The figure below gives a brief overview of the IP protocol.



Interwebbed communications

After each device is assigned with an IP address, it allows the cross communication with devices and the internet. The traffic is filtered through the router and off to the ISP it goes. Everytime a request is made through the internet (Think of visiting a website, streaming music, searching for an image, sending an email, etc) a connection must take place. Whenever there is a click of your mouse or keyboard, a request is made. There are many different protocols, ICMP, TCP, UDP, ARP, to name a few.     ClamAV

    ClamAv is simple and quick virus scanner, ClamAv can be installed with "apt-get install clamav"



    Another amazing feat of ClamAV is it does not have to scan the entire computer system, with the ability to scan directories and individual files, ClamAV gives a range of security, from a system wide scan to a sketchy file downloaded from the interwebs.



    Below are some of the hiccups I had during install. There are a couple steps to get this scanning service up and running. I have found that after the "apt install" a service named "freshclam" begins to run. This is a script to check to see if ClamAV is up to date and a quick configuratioin with the system hardware. After install if the command "ps aux | grep freshclam" is executed you will notice it running. The only way I could get freshclam to work is completely restart the freshclam service. "sudo pkill -15 -x freshclam" then "sudo freshclam" allowed for the update of ClamAV.

    Helpful ClamAV commands

    After creating a directory for ClamAV (sudo mkdir ~/Desktop/ClamScan) you have the abililty to scan and send all of the suspect files to that folder. Linux allows you can contain the suspect files in a quaratined folder (Don't forget to remove the permissions). With the proper persmissions set for this directory it isolates the files away from the system. I setup a Crontab in order for ClamAV to run at 3am once a week.


    The crontab above will run clamscan with -r which will remove the files. The next piece "--move=" will move the suspect files. "~/Desktop/ClamScan" will be the location to send the files and "/*" specifies ClamAV to scan the entire root directory.
